Notion → Astro → Statyczna strona

Cała integracja z Notion dzieje się raz, podczas budowania. Użytkownik nigdy nie rozmawia z Notion — dostaje gotowy, lekki HTML. Token zostaje bezpieczny, a strona ładuje się błyskawicznie na mobile.

Przepływ danych — krok po kroku

1
w Notion
(kiedy chcesz)

Baza w Notion źródło · token sekretny

Każda książka to wiersz w bazie Notion: tytuł, autor, okładka, ocena, treść recenzji. Edytujesz to wygodnie jak dokument — to Twój CMS. Dostęp chroni integration token, który NIGDY nie trafia do przeglądarki.

2
build
(na serwerze)

Astro pobiera dane build-time

Podczas astro build kod na serwerze odpytuje Notion API jednym fetch-em. Token czytany jest z .env — pozostaje na serwerze buildu, nie w wynikowym kodzie.

// src/lib/notion.js — wykonuje się TYLKO przy buildzie const res = await fetch('https://api.notion.com/v1/databases/.../query', { headers: { Authorization: `Bearer ${process.env.NOTION_TOKEN}` } }); export const books = mapNotion(await res.json());
3
build
(renderowanie)

Generowanie stron build-time

Astro renderuje komponenty .astro z tymi danymi do czystego HTML-a. Dla każdej książki powstaje osobny plik — np. /recenzje/acotar/index.html — przez routing getStaticPaths(). Okładki możesz przy okazji zoptymalizować (WebP/AVIF).

// src/pages/recenzje/[slug].astro export async function getStaticPaths() { return books.map(b => ({ params: { slug: b.slug }, props: { b } })); }
4
deploy
(CDN)

Publikacja na CDN statyk

Gotowy folder dist/ (sam HTML, CSS i obrazki) ląduje na statycznym hostingu — Netlify, Cloudflare Pages, GitHub Pages. Za darmo, globalnie, bez serwera do utrzymania.

5

Użytkownik na mobile runtime · 0 zapytań do Notion

Przeglądarka pobiera gotowy HTML prosto z CDN — żadnego odpytywania Notion, żadnego oczekiwania na API. Interaktywność (np. filtrowanie, sortowanie) działa jako pojedyncza „wyspa", która jako jedyna ładuje trochę JS-u.

Jak aktualizować treść?

↻

Bo dane „zamrażają się" przy buildzie: po edycji książki w Notion trzeba przebudować stronę. Przy projekcie hobby wystarczy ręczny redeploy jednym kliknięciem. Można też ustawić webhook z Notion → automatyczny rebuild albo prosty cron (np. raz dziennie). Treść zmienia się rzadko, więc to nie boli — a w zamian masz najszybszą możliwą stronę.
